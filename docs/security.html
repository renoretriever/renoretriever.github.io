Ping of Death(PoD) 攻撃

  • 悪意のある Ping パケットを大量に送りつけることで、サーバを停止させる攻撃手法

  • 攻撃に晒された場合、緊急対応として一旦全種類の Ping に応答しないようにする

    # sysctl -w net.ipv4.icmp_echo_ignore_all=1

  • 認証局(CA : Certification Authority, 認証機関)
    • クライアントとサーバーが双方ともに信頼する第三者機関がその立場であり、公開鍵が本物であることを証明する機関
    • 利用
      1. サーバーの運営者が認証機関に公開鍵を提出
      2. 認証機関はその提出された公開鍵にデジタル署名を施し、署名済の公開鍵を作る
      3. その署名済の公開鍵が公開鍵証明書(デジタル証明書)に収められる
    • 証明書を受け取った
  • SSL 証明書
    • いまさら聞けない、SSLサーバ証明書とルート証明書の関係
    • サイトチェッカー
      • 不要な中間証明書等も検知してくれるから優秀
        • https://www.ssllabs.com/ssltest/
        • 「Do not show the results on the boards」にチェック入れないと画面下のドメイン一覧にのってしまう
    • SSLサーバ証明書の種類と比較
      • DV (Domain Validation)
        • ドメイン認証
        • ドメインの管理権限をもとに発行される
      • OV (Organaization Validation)
        • 企業認証
        • 組織情報の審査を経て発行される
      • EV (Extended Validation)
        • 通信相手が実在する企業であるかを確かめる役割をもった証明書
        • ブラウザのアドレスバーの色が緑色になる
        • SSL 証明書の発行にて一定の基準に基づき、それをクリアして発行された証明書
        • EV 証明書ガイドライン
          • 企業の法的実体の確認
          • ドメインの所有権の確認
          • 担当者の身元証明と署名
  • クライアント証明書
    • サーバーが通信している相手が意図したクライアントであることを証明
    • 問題点
      • ユーザーにクライアント証明書をインストールしてもらう必要がある
      • コスト増
      • クライアントの実在を証明するだけで、ユーザーの実在を証明するユーザ証明書ではない
  • 証明書失効リスト(CRL)

  • 出力値のエスケープの不備による脆弱性

    • SQL インジェクション
    • OS コマンドインジェクション
    • HTTP ヘッダーインジェクション
    • ディレクトリ・トラバーサル
    • リモート・ファイル・インクルージョン

  • 設定や設計の不備による脆弱性

    • 強制ブラウジング
      • 公開することを意図していないファイルが閲覧可能な状態になっていること
        • ファイルの連番とかで推測される
      • 認証が必要なサイトであるけど直 URL を知っていれば認証を経ずに閲覧可能
    • 不適切なエラーメッセージ処理
      • Web アプリケーションのエラーがブラウザ上で表示されると攻撃者にとっては情報になり得る
      • PHP, DB のエラー
    • オープンリダイレクト
      • 指定した任意の URL にリダイレクトする機能
  • セッションアドプション(Session Adoption)
    • PHP や ASP.NET などに存在する未知のセッション ID を受け入れる機能
  • クリックジャッキング
    • 透明なボタンやリンクを罠となる Web ページに埋め込み、そのリンクをユーザにクリックさせることで意図しないコンテンツにアクセスさせる攻撃
    • UI Redressing
    • X-Frame-Options
