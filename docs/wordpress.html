WordPress

Important

  • WordPress をアップデートすると削除したファイル(xmlrpc.php など) は復活する

  • WordPress 本体やプラグインを更新するとファイルの権限が変わる可能性があるのでそこの調整が必要

  • 管理画面にアクセスすると Segmentation fault になる

    • eaccelerator が入っているか確認

    • 入っていてかつ有効になっている場合には、php がエラーはいて落ちる

    • 対象ディレクトリ配下の .htaccess にて下記を追加してあげる

      php_flag eaccelerator.enable 0
php_flag eaccelerator.optimizer 0

  • wp-config.php 設置後のインストールをコマンドラインから実行

    # curl -F "weblog_title={{ wordpress_title }}"  \
     -F "user_name={{ wordpress_user_name }}"  \
     -F "admin_password={{ wordpress_admin_password }}"  \
     -F "admin_password2={{ wordpress_admin_password }}"  \
     -F "admin_email={{ wordpress_admin_email }}"  \
     -H 'HOST: test.com'  \
     http://test.com/wordpress/wp-admin/install.php?step=2

ファイル所有者/権限

前提

  • 前提

    • 管理画面からアップデートできるようにするには WordPress 配下のファイル所有者が PHP 実行ユーザと同じである必要がある
      • ここでは apache にする
    • FTP user は ftptest とする
    • document root
      • /srv/example.com/

  • 想定

    • FTP パスワードなど聞かれずに、管理画面から更新ボタンを押下し、バージョンアップデートを実施できる
    • FTP にてファイルのアップロード/ダウンロードができる

  • 更新は wordpress 配下のファイル所有者権限と php 実行ユーザの権限が同じでないと更新されないので、合わせて上げる必要がある

    • define('FS_METHOD','direct'); つけていても上記があってないとだめ

作業実施

  • WordPress 配下の権限を apache(PHP 実行ユーザに変更)

    sudo chown -R apache:apache /srv/example.com/

  • uesr, group に書き込み権限を付与

    sudo chmod -R ug+w /srv/example.com/

  • apache サブグループへの追加

    # usermod -aG apache ftptest

  • FTP サーバの設定変更(vsftpd 想定)

    local_umask=002

試す

  • 管理画面から更新ボタンを押して見る
    • FTP パスワード聞かれなければ ok
  • FTP ユーザ使ってコンテンツの更新
    • 正常にできることを確認
    • ファイルの権限が想定であることを確認
  • コンテンツ更新した後に再度管理画面から更新してみる
    • FTP パスワード聞かれなければ ok
