Kerberos

  • 認証および暗号化システム

  • 構成

    • レルム(realm)と呼ばれる管理領域を、プリンシパル(レルム内のすべてのクライアント及びサーバの総称) と、信頼される第三者である kdc(key distribution center) で構成する

  • 特徴

    • レルムの範囲内において、kdc がサーバへのアクセスを要求するクライアントを認証し、共通鍵で暗号化された有効期限付きのチケットを発行
    • クライアントは kdc から発行されたチケットを用いて目的のサーバにアクセスする
    • レルム内では、サーバとクライアント間での二者間認証は一切行われず、kdc を介した三者間で認証が行われる
    • 次の三つのサーバ(機能) から構成される
      • 1 台で賄うことが多い
      • 認証サーバ(authentication server : as)
        • kdb に保存されている認証情報にもとづいてユーザを認証し、チケット交付チケット(ticket granting ticket : tgt) を発行
      • チケット交付サーバ(ticket granting server : tgs)
        • チケット交付チケットをもつユーザに対して各サービスを利用するためのチケットを発行する
      • データベースサーバ(kerberos database : kdb)
        • 各ユーザの認証情報や共通鍵などを保存する
