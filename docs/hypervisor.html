ハイパーバイザー

基礎

  • 通常のホスト OS の代わりにハードウェア上で動作し、アプリケーションの代わりに仮想マシンを実行
  • ハードウェアとゲスト OS の橋渡しをする役割
  • 仕組み
```
---------------------------------------
|   仮想マシン A   |   仮想マシン B   |
| アプリケーション | アプリケーション |
|    ゲスト OS     |    ゲスト OS     |
| 仮想ハードウェア | 仮想ハードウェア |
---------------------------------------
|           ハイパーバイザー          |
---------------------------------------
|          物理ハードウェア           |
---------------------------------------
```
  • メリット
    • 仮想マシンのリソースとホストシステムのリソースを対応付け、仮想マシンがあたかも独立したホストシステムのように振る舞えるようにする
    • => ホストシステム上で動作する複数の仮想マシンがお互いに干渉しない
    • ホスト OS 型よりもパフォーマンスが高い

完全仮想化 / 準仮想化

準仮想化(Para-Virtualization)

  • 仮想マシン上で動かすゲスト OS のコードに改変(修正) が必要となるタイプ
    • 通常環境
      • アプリケーションからのシステムコールは OS のカーネルが処理
    • 準仮想化環境
      • ゲスト OS からのシステムコールをハイパーバイザーが処理
        • => ハイパーバイザーコール

  • OS の一部を改変して動作させる必要があり、OS 自身は自分が仮想化されていることを認識している

  • 仮想マシンの性能を向上させるための技術

  • 物理マシンで動作する OS と同じ OS を仮想マシンで実行する完全仮想化を実現する技術が HVM(HardwareVirtualMachine) ですが、完全仮想化にはストレージやネットワークの I/O などの性能が劣化する問題がある

  • これらの I/O のみを仮想化に最適化するための手段が準仮想化で

  • 準仮想化の仮想マシン上で準仮想化に対応したカーネルを利用するほか、完全仮想化の仮想マシンに準仮想化対応デバイスドライバをインストールすることで性能を向上させる方法などがある

  • 完全仮想化＋準仮想化対応デバイスドライバの組み合わせで動作させることが多く、Linux カーネルにもいくつかの準仮想化ドライバがあらかじめ組み込まれています

完全仮想化(Full Virtualization)

  • ゲスト OS のコードに一切手を加える必要がなく動作させることができる

  • デメリット

    • 準仮想化に比べて処理オーバーヘッドが大きく、パフォーマンス劣化する

  • 仮想ハードウェアによる ハードウェアエミュレーション と呼ばれる仕組みを用いて I/O 命令や割り込み、メモリ管理など、ハードウェアに対する処理をソフトウェアレベルで忠実に再現する

  • ゲスト OS の特権 CPU 命令をハイパーバイザー上の仮想ハードウェアがエミュレートする

種類

  • モノリシック型

    • デバイスなどのドライバーをハイパーバイザー内に含め、かつその多くが単体で、管理 OS 的なものを持たない

  • マイクロカーネル型

    • ハードウェアごとに管理 OS を設け、デバイスなどのドライバーが含まれる

製品

  • Hyper-V

    • Microsoft 社製品
    • 管理 OS として Windows Server

  • VMware ESXi

    • VMware 社製品
    • 最小構成では管理 OS などもない

  • Xen / Citrix XenServer
