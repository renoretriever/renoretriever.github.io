Web

  • URL エンコーディング(% エンコーディング)

    • URL に使えない文字を扱えるようにするためのエスケープ処理

  • URI(Uniform Resource Identifires)

    • リソースの識別
    • URL(Uniform Resource Locator)
      • リソースの場所を示す
      • http://user:pass@www.example.jp:80/dir/index.htm?uid=1#ch1
        • http:
          • スキーム名
          • data:, javascript: などなど
        • user:pass
          • 資格情報
          • オプション扱い
        • www.example.jp
          • サーバ名
          • ドメイン名 or IP アドレス(v4, v6)
        • :80
          • サーバのポート
        • dir/index.htm
          • 階層的なファイルパス
        • uid=1
          • クエリ文字列
        • #ch1
          • フラグメント識別子
          • リソース内のサブリソース
    • URN(Uniform Resource Name)
      • リソースの名前を示す
      • 例
        • ISBN を利用した URN
        • urn:isbn:1111111111

Web サーバ

クライアント PC から Web サーバまでの

  • 構成
    • Web ブラウザ => Web サーバ(Apache HTTP Server) => AP サーバ => DB サーバ
  1. Web ブラウザにて URL を入力し検索実行。HTTP リクエストを実行。
  2. DNS にて名前解決を行い目的のグローバル IP アドレスを取得し、Web サーバに対して HTTP リクエストを送信
  3. Web サーバの httpd がリクエストを受け取り、ディスクから読み込むか、AP サーバに問い合わせするか判断
  4. 接続元にリクエストを返す

キャッシュ

  • プロキシサーバーやクライアントのローカルディスクに保存されたリソースのコピーのこと

  • メリット

    • サーバーとの通信量の削減
    • ネットワークに近いところから取得できる

  • クライアントサイドキャッシュ

    • ブラウザやプロキシーサーバがキャッシュ
    • <meta> タグを使ってコンテンツ単位でキャッシュを指定できる

  • キャッシュヘッダー

    • HTTP/1.0
      • Expires や Pragma といったヘッダーでキャッシュデータを制御
    • HTTP/1.1
      • Cache-Control ヘッダ
    • 通常は Cache-Control のみで OK だけど、ブラウザやプロキシサーバによっては HTTP/1.1 に対応してない場合があるため、Expires/Pragma も併用する

    • ETag
    • 強い ETag 値と弱い ETag 値
      • 強い
        • エンティティがわずかに違っても必ず値は変化
      • 弱い
        • リソースが同じものであることということしか示さない
        • 意味上異なったリソースとして差異がある場合にのみ ETag 値が変化する
        • 先頭に「W/」 が付く
