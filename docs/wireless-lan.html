無線 LAN

  • 機器構成

    • ルータ機能と同居するタイプ
    • ルータ機能をもたないアクセスポイント

  • 無線規格

    • IEEE 802.11 a/b/g/n/ac
    • IEEE 802.1X
      • WPA や WPA2 では無線アクセスポイントに接続する無線端末を認証するのに利用
      • 無線アクセスポイントから無線端末に暗号化鍵を配布できるようにしている
        • WPA や WPA2 では利用者認証のほか、動的に更新される暗号化鍵を用いて暗号化通信を行なうことができる

  • 給電方式

    • 通常の AC アダプタ
    • PoE(Power over Ethernet)

  • 伝送速度(実効速度)

    • 11b
      • 4 Mbps 程度
    • 11a/11g
      • 22 Mbps 程度
    • 11n
      • 80 Mbps 以上
    • 11ac
      • 400 Mbps 以上

  • 接続数

  • 通信距離と干渉

  • セキュリティ

    • 接続機器の制限
      • SSID の設定
        • アクセスポイントとクライアントの SSID 設定値が一致する場合のみ接続できるようにする
      • MAC アドレスによろ制限
    • 通信の暗号化
      • WPA2-AES
      • WPA2-PSK/AES
  • アクセス制御
    • CSMA/CA
      • 送信前に他に通信しているホストが無いことを確認してからデータを送信
  • IEEE802.11a
    • 5 GHz 帯
    • 伝送速度 54 Mbps
    • 特徴
      • 高速な伝送速度と、電波干渉を受けにくいというメリットがあったが国内では普及しなかった
    • 屋内では使用不可
  • IEEE802.11b
    • 2.4 GHz
    • 伝送速度 11 Mbps
    • 特徴
      • 安価な対応製品が多くノート PC や携帯ゲーム機への搭載がすすんだことから広く普及
      • 2.4 GHz 帯は家電製品との電波の干渉で速度が低下する場合がある
  • IEEE802.11g
    • 2.4 GHz
    • 伝送速度 54 Mbps
    • 特徴
      • IEEE802.1b に対する上位互換
      • 移行が容易なことから現在の主流
  • IEEE802.11n
    • 2.4 GHz / 5 GHz
    • 伝送速度 600 Mbps
    • 特徴
      • MIMO(Multiple Input Multiple Output) 方式により複数のアンテナを組み合わせてマルチストリーミングによる高速化と通信の安定化を達成している
      • チャンネルボンディングと呼ばれる 2 つのチャンネルを組み合わせる方式により、1 チャンネル時のおよそ 2 倍の速度での通信が可能
      • 国内では電波法上の制約から伝送速度は 300 Mbps に抑えられている

セキュリティ対策

  • WEP(Wired Equivalant Privacy)

  • WPA(Wi-Fi Protected Access)

  • IEEE802.1X

  • ESSID

    • 複数のアクセスポイントをグループ化する識別符号(ID)
    • ESSID 隠蔽昨日
    • ESSID を「ANY」に設定したクライアントからの接続を拒否する機能を備えるもの
    • ステルス化

  • MAC アドレスフィルタリング - 登録した NIC の MAC アドレス以外を接続できなくする機能 - 脆弱性 - パソコンの MAC アドレスは簡単に書き換えることができ、MAC アドレス自体が平文で流れているので簡単に偽装されてしまう

  • VPN の併用 - IPSec を使用して、暗号化された VPN で通信を行なうようにする対策がある - 無線 LAN 同士が IPSec を使わずに通信できていしまうことや、だまされて偽のアクセスポイントと通信してしまう等のクライアント側の対策が必要

セキュリティ強化策

  • IEEE 802.1X 規格に基づく認証機能の追加

    • EAP 採用

  • TKIP(Temporal Key Integrity Protocol) により暗号化機能(鍵管理) 及び完全性チェック機能を強化した WPA

    • TKIP は下記のようにして WEP の脆弱性を克服
      • 共有キー(WPA-PSK) のサイズを 128 ビットに拡張
      • IV のサイズを 48 ビットに拡張
      • WPA-PSK と IV, MAC アドレスからハッシュ値を求め、暗号鍵として使用
      • ユーザ認証後に認証サーバから最初の鍵を交付
      • 一定量のパケットを送信する毎/一定時間ごとに暗号鍵を更新することが可能
      • MIC(Message Integrity Code) の採用により、完全性チェック機能を強化

  • AES の採用により暗号化機能(アルゴリズム) を強化した WPA2

    • WPA2
      • AES を採用した CCMP(Conter-mode with Cipher Block Chaining Message Authentication Code Protocol) という暗号方式が実装
      • WPA2-Enterpriseでは，IEEE802.1Xの規格に沿った利用者認証及び動的に配布される暗号化鍵を用いた暗号化通信を実現できる
      • パーソナルモード
        • WPA2-personalは、個人や小規模企業などの認証サーバを用意できないネットワーク向けの方式で、端末に設定された事前共通鍵(PSK)を用いて認証を行う
      • エンタープライズモード
        • WPA2-Enterpriseは、認証にIEEE802.1Xを用いる方式
        • IEEE802.1Xでは、クライアント－アクセスポイント－認証サーバの3者間で認証情報がやり取りされます
        • 認証に成功すると、認証サーバはPMK(Pairwise Master Key)と呼ばれる暗号化鍵の種を動的に生成し、アクセスポイントと認証された端末に配布
        • 暗号化鍵はこのPMKから生成されるのでセッションごとに異なる仕組みになっている
  • War Driving
    • 車に SSID 名や WEP の使用、不使用、アクセスポイントの MAC アドレス等を調査するソフトをインストールしたノート PC を積んで、街中を走り回り、無線 LAN のアクセスポイントの情報を収集して回ること
  • LLDP(Link Layer Discovery Protocol)
    • 定期的に隣接する機器の情報を収集するプロトコル
    • 情報を提供する側の機器は LLDP パケットをマルチキャストアドレス宛てに定期的に送信し、情報を収集する側の機器はLLDPパケットを受信することにより情報を収集します。
    • 受信したLLDPパケットに含まれる情報は、装置のリモートテーブルに保持されます。
