パスワードクラック

  • OS やアプリケーションプログラムに設定されたパスワードを破ることを目的として、何通りものパスワードや流出したパスワードリスト等を用いてパスワードを解読したり、ログインを思考したりする手法

種類と実行方法

  • オンライン攻撃

    • ターゲットとなるシステムにネットワークを介して実際にログインを試みながら行う方法

  • オフライン攻撃

    • パスワードが保存されたファイルを何らかの手段で入手し、それを攻撃者のローカル環境で解読する方法

  • 推測によるパスワードクラック

    • ユーザ ID や利用者の情報から攻撃者がパスワードを推測し、実行する方法
    • 例
      1. admin
      2. admin01
      3. admin99

  • 辞書ファイルを用いたパスワードクラック

    • パスワードに使われそうな文字列が大量に登録されてたファイルを用いて順次試していく方式
    • 辞書攻撃
    • 例
      • test
      • sys
      • root
      • 123456
    • 対策
      • ランダムな値でパスワードを設定する

  • 総当りによるパスワードクラック

    • 特定の文字数、文字種で設定され得るすべての組み合わせを試す方式
    • ブルートフォース攻撃
    • 対策
      • ログイン試行回数に制限を設ける

  • レインボーテーブル を用いたパスワードクラック

    • レインボー攻撃
      • パスワードがハッシュ値で保管されている場合に、あらかじめパスワードとハッシュ値の組み合わせリスト(レインボーテーブル)を用意しておき、そのリストと突き合わせてパスワードを推測し、不正ログインを行なう
    • time-memory trade-off テクニック
      • メモリの使用量が削減できる代わりにプログラムの速度が低下する、または逆に、計算にかかる時間を削減できる代わりにメモリの使用量が増える、という状況のこと
    • ソルトを利用して対策を行う
      • パスワードからハッシュ値を求める際に、パスワードに付加する文字列のこと
      • ユーザごとにランダムな文字列であることと、ある程度の長さ(少なくとも 20 文字程度) であることが求められる
      • ソルトを使用することで同じパスワードであっても出力されるハッシュ値が変わるため、ハッシュ値から元のパスワードを特定することが困難になる

  • 線形解読法

    • 平文と暗号文と鍵の関係を代数式に表して数学的に鍵を見つけ出す

  • 関連鍵攻撃

    • RC4の解読攻撃に使用される
    • 平文の一部分の情報と暗号文の一部分の情報との間の統計的相関を手掛かりに鍵を見つけ出す

  • 差分解読法

    • 入力値の差分が出力値の差分にどのような影響を与えるかを分析して解読を試みる
    • 平文を一定量変化させたときの暗号文の変化から鍵を見つけ出す

対策

  • 予防・防止

    • ワンタイムパスワード方式、バイオメトリクス認証システムの導入
    • アカウントのロックアウト方式
    • パスワードからハッシュ値を求める際にソルトを使用する
    • ログイン失敗をログに残す

  • 検知・追跡

    • IPS/IDS
    • ログイン失敗ログの検知

  • 回復

    • OS クリーンインストール

リバースブルートフォース攻撃

  • パスワードを固定して何通りものユーザ ID の組み合わせを試行する手法

パスワードリスト攻撃

  • 他のサイトで取得したパスワードのリストを利用して不正ログインを行なう攻撃
  • パスワードの使い回しが要注意
  • 対策
    • インターネットサービスごとに異なるパスワードを使用する
