パフォーマンス

  • アルゴリズム
  • レスポンス / スループット
    • レスポンス
      • 1 処理あたりの所要時間
      • クライアント側の視点
    • スループット
      • 単位時間あたりの処理量
      • サーバ側の視点
    • パーセンタイル

パフォーマンス分析

  • 必要な情報を取る

    • サマリ形式
      • 一定期間の情報を合計もしくは平均を見せてくれる
      • iostat
      • netstat
        • ss
      • sar
      • vmstat
      • perf
    • イベント記録形式
      • 個々のイベントを逐次記録する方式
      • tcpdump
      • strace
      • デメリット
        • データが大きくなる
        • 負荷が大きくなる
    • スナップショット形式
      • その瞬間の状態を記録
      • 定期的に実行する必要がある
      • ps
      • top
      • pstack

  • 最低でも 1 分間隔で取得する

  • 待ち行列理論

Web サーバのパフォーマンス分析

  • アクセスログを出力
    • リクエストの応答時間を出す

DB サーバのパフォーマンス分析

  • スローログを出す
    • スローログはあくまで結果なので、それを引き起こした原因がもしかしたら OS 側にあるかもしれないので要注意
  • CPU or I/O 待ちかを見極める

ストレージのパフォーマンス分析

  • スループット or IOPS を見る
