ファイアウォール

基礎

  • インターネットと接続する出入り口に設置する最初のセキュリティ対策

機能

  • 隘路(あいろ, CHOKE POINT)

    • インターネットとの接点を 1 箇所にし、集中して防御

  • スクリーン(ルータ)

    • インターネット(外部)と社内 LAN をカーテンで仕切るように分離

  • 検閲(パケットフィルタリング)

    • 通行の許可、遮断を制御
    • 予め許可された通信だけを通すように

  • 隠蔽(NAT, 不要な応答をしない設定)

  • 記録(ログ)

    • 入出力の通信を記録し、不審なアクセスの発生や兆候を証跡として残します

  • 代理(プロキシ)

  • アドレス変換機能

    • NAT
    • NAPT(IP マスカレード)

種類

  • パケットフィルタリング・ルータ

    • 送信元 IP アドレス、送信元ポート番号 、 あて先 IP アドレス、あて先ポート番号 を元にアクセス制御リストを作成し制御判断

  • サーキットレベル・ゲートウェイ(3, 4, 5 層)

    • プロトコルを解釈せずにクライアントとサーバ間で回線(サーキット)を生成するゲートウェイで、現在はほとんど使用されていない
    • SOCKS

  • アプリケーションレベル・ゲートウェイ(3 〜 7 層)

    • 全てのパケットを一度ゲートウェイが受け取り、すべての段階で通信内容がルールにそっているか確認し、同じ内容のパケットを作成して自ら送信元にして宛先を送り出します。
    • 欠点
      • いったん全ての階層で処理を行なうことから処理時間がかかり通信のオーバーヘッドが大きくなる

  • ステートフルインスペクション型ファイアウォール

    • 通信の状態情報を収集し、そこから次に来るパケットを予測し、矛盾がないかを確認してアクセス制御を行います
    • 常に動的にアクセス制御リストを作成・更新
    • IP スプーフィング などの偽装した通信を遮断
      • サーバなどで利用する IP アドレスを偽造して、他のサーバなどになりすますこと
      • DoS 攻撃や DNS キャッシュポイズニング攻撃などと合わせて、攻撃者の身元を隠すために利用

  • パーソナル・ファイアウォール(一般的な分類)

    • 主に個人 PC に導入

  • WAF(Web アプリケーションファイアウォール)

構成

  • DMZ
    • ファイアウォールに守られつつも、社内からも社外からも安全にアクセスできる緩衝地帯

限界

  • 正常な通信と区別できない場合
    • ウィルスやワーム
    • セキュリティホールを利用した攻撃
    • DoS 攻撃
    • 暗号化された通信
