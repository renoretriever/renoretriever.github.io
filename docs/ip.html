IP(Internet Protocol)

  • 目的のホストまでパケットを届けてくれる
  • レイヤー 3 / ネットワーク層
  • IPv4
  • IPv6
  • 目的
    • IP アドレスによる最終宛先へのデータ転送
    • ルーティングテーブルを用いてルーティングを行う

IP パケット

  • 構造

    --------------------------------------------------------------------------------
| Ethernet ヘッダ | IP ヘッダ | オプション(4 x n バイト) | IP ペイロード | FCS |
--------------------------------------------------------------------------------

  • エラー処理は ICMP を利用

IP ヘッダ(20 バイト)

0           3 4   7 8   13 14 15 16       18 19                      31(bit)
----------------------------------------------------------------------
| バージョン | IHL | DSCP | ECN |             パケット長             |
----------------------------------------------------------------------
|            識別子             | O | F | M | フラグメントオフセット |
----------------------------------------------------------------------
|   TTL   |   プロトコル番号    |         ヘッダチェックサム         |
----------------------------------------------------------------------
|                          始点 IP アドレス                          |
----------------------------------------------------------------------
|                          終点 IP アドレス                          |
----------------------------------------------------------------------
|                       オプション                      | パディング |
----------------------------------------------------------------------

  • バージョン

    • IP バージョン

  • IHL(ヘッダ長 : Internet Header Length)

    • IP ヘッダの長さ

  • DSCP(Differentiated Service CodePoint)

    • DiffServ を運用するときに使われる
      • => IP に QoS を導入することができる

  • ECN(Explicit Congestion Notification)

    • 途中のルータがふくそう状態かどうかを通知するために使われる

  • パケット長

    • IP データグラム全体のバイト長

  • 識別子

    • フラグメントを復元するときの識別子として使われる

  • フラグ

    • IP パケットの分割処理を制御する 3 ビットのフィールド
    • O
      • 未使用
      • O にしなければならない
    • F
      • 途中のルータに分割してよいかを指示
      • 0
        • 分割可
      • 1
        • 分割不可
    • M
      • 運んでいるデータがフラグメントの途中か最後かを示す
      • 0
        • 最後のフラグメント
        • フラグメントしていない
      • 1
        • 途中のフラグメント

  • フラグメントオフセット

    • 分割されたデータが、オリジナルデータのどこに位置しているかを意味する

  • チェックサム

    • IP パケットがネットワーク中を転送されている間に、ノイズなどによって IP ヘッダが壊されていないことを保証するための情報を格納する場所
    • 保証対象
      • バージョン
      • ヘッダ長
      • DSCP
      • ECN
      • パケット長
      • 識別子
      • フラグ
      • フラブメントオフセット
      • 生存期間
      • プロトコル番号
      • 始点 IP アドレス
      • 終点 IP アドレス
  • IP での通信
    • IP の通信では MAC アドレスを利用して通信を行う
      • LAN 外のコンピュータと通信を行う場合には、中継先の MAC アドレスを使う
    • ARP を利用して、宛先 IP アドレスをもとに MAC アドレスを調べる

制限

  • パケットが目的地まで到達する保証はない
  • 送信したパケットが順番通りに届く保証はない
  • 1 つのパケットが複数に増える可能性がないとは言えない
  • ペイロードが壊れていないことを保証しない
  • 一度に運べるペイロードの最大サイズは 65515 バイトに制限
    • => MTU よりも大きなデータを送信しなければならなくなった場合、IP は 1 つの IP パケットを複数のデータリンクのフレームに分けて送信
      • => IP フラグメンテーション
