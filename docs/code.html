暗号

基礎

  • 何らかのデータを、そのまま読むことの出来ない形式に変換すること

  • 暗号には、暗号化する際の手順(アルゴリズム) と、その際に用いる固有の値(鍵) が必要になる

    • シーザ暗号
      • 元のアルファベットを 3 文字ずらすことで暗号文を作成
      • アルゴリズム
        • 文字をずらす
      • 鍵
        • 3 文字

  • 平文

    • 暗号化されていない元の文

  • 暗号化

    • 元のデータを暗号にする操作

  • 復号

    • 暗号化されたデータを元に戻す操作

  • 暗号アルゴリズム

    • どのなような暗号文を生成するかという仕組み

  • 暗号技術

    • 情報などの秘匿
    • 情報などの認証(メッセージ認証) 及び署名
      • ハッシュ関数
      • ディジタル署名

主な暗号方式

古典暗号

  • 転置式暗号
    • 一定の規則に基いて文字の順番を入れ替えて暗号文を生成
    • password => drowssap
  • 換字式暗号
    • 一定の規則に基いて別の文字や記号などに変換
    • drowssap => espxiibq

現代暗号

  • 共通鍵暗号方式

  • 公開鍵暗号方式

  • ハッシュ

  • ハイブリッド式

    • 公開鍵暗号方式(データの暗号化に用いた共通鍵を安全に渡す) + 秘密鍵暗号方式(データの暗号化の処理が早い)

  • 共通鍵暗号方式と公開鍵暗号方式の比較

  • 共通鍵暗号方式と公開鍵暗号方式のメリット・デメリット

  • CRYPTREC

ハッシュ関数、MAC、フィンガプリント

[ハッシュ関数]

MAC(Message Authentication Code : メッセージ認証コード)

  • 通信データの改ざん有無を検知し、完全生を保証するために通信データ

  • 完全性を保証するために通信データから生成する固定長のコード

  • CMAC(Cipher-based MAC)

    • ブロック暗号

  • HMAC

フィンガプリント

  • ディジタル証明書や公開鍵、メールなどの電子データが改ざんされていないことを証明するために使用するデータ
