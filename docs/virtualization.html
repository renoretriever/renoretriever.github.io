仮想化技術

基礎

仮想化

  • 物理的なハードウェアリソース(CPU やメモリ、ストレージなど) をユーザから隠蔽し、代わりとなる論理的なリソースを提供するための技術・仕組み

  • 仮想マシン(VM)、仮想ホスト、仮想サーバなどと呼ばれる

  • メリット

    • システムの柔軟性と拡張性を向上
    • コスト削減
      • サーバの設置場所
      • 物理サーバの調達や運用コスト

  • デメリット

    • 性能劣化
  • エミュレータ
    • 異なるアーキテクチャのコンピュータ用のプログラム命令を解読しながら実行するマイクロプログラム
    • 例
      • パスコン上で動作するスマホのエミュレータ

仮想化種類

  • 仮想化は主にソフトウェアによって提供される仕組みであり、サーバを仮想化するためのソフトウェアを 仮想化ソフトウェア や 仮想マシンモニタ(VMM : Virtual Machine Monitor) という

ホスト OS 型

  • 物理的なハードウェア上に OS をインストールし、その OS 上で仮想化ソフトウェアと仮想マシンを動かす方式

  • ホスト OS

    • 物理マシン上で直接動作する OS

  • ゲスト OS

    • 仮想マシンにインストールされる OS

  • 基本的な仕組み

    ---------------------------------------
|   仮想マシン A   |   仮想マシン B   |
| アプリケーション | アプリケーション |
|    ゲスト OS     |    ゲスト OS     |
| 仮想ハードウェア | 仮想ハードウェア |
---------------------------------------
|          仮想化ソフトウェア         |
---------------------------------------
|              ホスト OS              |
---------------------------------------
|          物理ハードウェア           |
---------------------------------------

  • 仮想化ソフトウェア

    • VMWare Workstation
    • VirtualBox

ハイパーバイザー型

  • ハイパーバイザー型

コンテナ型

  • カーネル上で動作しているプロセスをコンテナという単位で分離することで、別々の環境として利用できるようにする技術

  • ホスト OS としてインストールされた Linux カーネルが同じであれば、それぞれのコンテナを異なるディストリビューションの Linux OS として動かすことができる

  • カーネルは単一のものを利用するため、メモリ使用量が少なく、性能劣化もほとんど無いのがメリット

  • 隔離環境内ではあたかもホスト OS を自分だけが専有してるかのように外部環境とは独立してアプリケーションを実行できる

  • 仕組み

    ----------------------------------------------
|      コンテナ A      |      コンテナ B     |
|   アプリケーション   |   アプリケーション  |
----------------------------------------------
| コンテナエンジン(コンテナ管理ソフトウェア) |
----------------------------------------------
|                  ホスト OS                 |
----------------------------------------------
|              物理ハードウェア              |
----------------------------------------------

  • 仮想化ソフトウェア

    • OpenVZ
    • LXC
    • Linux VServer
    • Docker
    • Oracle Solaris Zones
  • ハイパーバイザ型/ホスト OS 型のハイブリッド
    • KVM
      • ハイパーバイザー型に近いが、仮想マシンが QEMU を利用していることや、仮想マシンがプロセスとして動作して見えることなどから、ハイパーバイザー型とホスト OS 型の中間ぐらいのハイブリット型とでも言うべき構造

HVM は完全仮想化と呼ばれる仮想マシンを実行し、この場合ゲストOSは通常と同じものが使用できます。PVは準仮想化と呼ばれ、OSにパッチを当てることで仮 想化環境に最適化を行いますが、オープンソースのOSか、ドライバのみ準仮想化するなどの方法をとる必要があります。

  • オーバーレイ
    • 主メモリにて処理ができないほどのアプリケーションがあった場合、対象アプリケーションを求められたサイズに分割し、必要に応じて主メモリへロードして処理を行うという機能

ハードウェアレベル仮想下

  • 仮想化ソフトはサーバーハードウェアを疑似的に再現した仮想マシンを提供

    • CPU, memory, 入出力装置などのハードウェア資源をソフトウェア的に再現

  • ユーザは仮想マシン上に通常のサーバを利用するのと同様にゲスト OS をインストールして、その上でアプリケーションを可動させる

                        | アプリケーション | アプリケーション |
                    |     ゲスト OS    |    ゲスト OS     |
                    |     仮想マシン   |    仮想マシン    |
| アプリケーション  |             仮想化ソフト            |
|                       ホスト OS                         |

  • メリット

    • ホスト OS と異なるゲスト OS を自由に選択・利用できる
    • ホスト環境とゲスト環境の分離度が高く、セキュリティ面では安全性が高い

  • デメリット

    • ハードウェアの再現やゲスト OS が必要になるため、利用資源量が多く、性能が出にくい
    • ゲスト OS の管理などが必要で管理コストが高い

マイグレーション

  • 移行作業

  • P2V(Physical to Virtual)

    • 物理サーバから仮想サーバへ
  • V2V(Virtual to Virtual)
    • 仮想化プラットフォーム間における仮想マシンのマイグレーション
  • ライブマイグレーション
    • ある物理サーバ上で稼働している仮想マシンを、OS やソフトウェアを停止させることなく別の物理サーバに移し替え、処理を継続させる技術
    • 切り替えによるダウンタイムはほとんどゼロで、移動前の処理やセッションが全て引き継がれるため可用性を損なうことがありません
  • バイナリトランスレーション
    • 仮想マシンから発行されたカーネルモードの CPU 命令を仮想マシンのユーザモード命令として処理し、その結果を仮想マシンの OS にわたす方法
    • ハードウェアの完全なエミュレーション
