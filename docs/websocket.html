WebSocket

  • Ajax や Comet を使った技術は HTTP をベースにしているため HTTP が持つボトルネックは解消されてない。WebSocket は新しいプロトコルと API によってその問題を解決するための技術として開発
  • 特徴
    • サーバープッシュ機能
    • 通信量の削減
  • ハンドシェイク・リクエスト
    • Upgrade ヘッダーを使って HTTP => WebSocket プロトコルに変更することを要請
    • Sec-WebSocket-Key: xxxxx にはハンドシェイクに必要なキーが格納される
    • Sec-WebSocket-Protocol: chat, superchat には使用するサブプロトコルが格納
  • ハンドシェイク・レスポンス
    • 101 Swiching Protocols で返す
    • Sec-WebSocket-Accept: xxxxx は Sec-WebSocket-Key の値から生成された値が格納
