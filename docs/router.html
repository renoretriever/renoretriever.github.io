ルーター

基礎

  • ネットワークを区切り、相互接続する機器。ネットワーク間のデータ転送を行う
    • WAN と LAN の仲介
    • LAN に異なる役割を持った複数のネットワークを作成し、それらが干渉しあわないように分断
  • IP ヘッダーを見てデータ転送を行う
  • ソフトウェアで制御
    • L3 スイッチ はハードウェアで制御
  • 昔はゲートウェイと呼ばれていた
  • MTU に合わせて IP パケットの分割処理は行うが、再構成処理 は行わない

ルーティング(経路制御)

  • ルータによるデータ転送
  • パケットがループするのを防止するため TTL を 1 減らす
  • ルーティングテーブル にはデータを転送するネットワーク情報が登録されており、作成されている必要がある

最適ルートの学習

  • ルーティングテーブルに登録されている ルート情報 の主な構成要素
  1. 情報源
    • 直接接続
      • ルータが直接接続しているネットワークのルート情報
      • 特別な設定は必要なく、インタフェースを有効にすると自動的に設定した IP アドレスに対応するネットワークアドレスのルート情報が登録される
    • スタティックルート
      • ユーザが手動でルーティングテーブルに登録
    • ダイナミックルーティング
      • ルータ同士が情報を交換しながらネットワークの構造を知る方法
      • ルーティングプロトコル
        • AS 内部の経路制御(IGP : Interior Gateway Protocol)
          • RIP
          • OSPF
        • AS 間の経路制御(EGP : Exterior Gateway Protocol)
          • BGP
  2. ネットワークアドレス/サブネット
    • ルーティングする送信先のネットワークを示す情報
  3. メトリック
    • ルータから目的のネットワークまでの距離を数値化したもの
    • 物理的な距離ではなく、ネットワーク的な距離
  4. アドミニストレーティブディスタンス(シスコ製ルータ)
  5. ネクストホップアドレス
    • ホップ はルータを指す
    • 次に転送すべきルータの IP アドレス
    • 原則的には、ルータと同じネットワーク内の他のルータの IP アドレス
  6. 出力インタフェース
  7. 経路時間

データの転送

  • デフォルトルート(デフォルトゲートウェイ)
    • 目的の送信先が見つからなかった場合にパケットを送るアドレス
    • 0.0.0.0/0 , default
  • ロンゲストマッチ(最長一致ルール)
    • ルータが経路選択する際の規則の一つ
    • ルーティングテーブルの中に、パケットの宛先ネットワークアドレスと一致するエントリが複数ある場合、より長いネットワーク部に一致したものを選ぶという規則
