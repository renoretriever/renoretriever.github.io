冗長化

  • 障害が発生しても予備の機器でシステムの機能を継続できるようにすることを指す
    1. 冗長化を想定する
    2. 予備の機材を準備する
    3. 運用体制の整備
  • 仕組み
    • 負荷分散
    • 死活監視
    • マスターの決定
    • フェイルオーバー

Web サーバの冗長化

  • 負荷分散(ロードバランス)

    • 複数台のサーバに処理を分散させてサイト全体のスケーラビリティを向上させる手法

  • DNS ラウンドロビン

    • DNS を利用して 1 つのサービスに複数台のサーバを分散させる手法
    • 問題点
      • サーバの数だけグローバル IP が必要
      • 均等に分散されるとは限らない(キャッシュ機能)
      • サーバがダウンしても気づかない

  • ロードバランサー

ネットワーク機器の冗長化

  • L2 スイッチの冗長化

    • カスケード
      • スイッチ間をクロスケーブルなどでつなげばスイッチをまたいで通信を行うことができる
    • L2 トランク
    • リンクアグリゲーション
      • コンピュータとスイッチングハブの間、または 2 台のスイッチングハブの間を接続する複数の物理回線を論理的に 1 本の回線に束ねる技術

  • L3 スイッチの冗長化

  • スパニングツリープロトコル(STP)

サイトの冗長化

  • DR
  • GSLB

other

  • コールドスタンバイ

    • 現用機が故障したら、予備機を手動で接続する
    • 注意点としては、現用機と予備機の設定は同じにしておく必要があるということ

  • ホットスタンバイ

    • 両方の機器を常に稼動させておき、常に同じ状態に保っておく運用形態

  • フェイルオーバー

    • 現用機に障害が発生した際に、自動的に処理を予備機に引き継ぐこと
    • VIP の割り当てを行い、その VIP を切り替えることでフェイルオーバーを実現

  • ヘルスチェック

    • ICMP 監視
      • WEB サービスがダウンしたかどうかはチェックできない
    • ポート監視
      • 過負荷状態で応答ができなくなっていたり、エラーを返していることは検知できない
    • サービス監視
      • HTTP リクエストなどの発行
      • 場合によっては、サーバに負荷をかけることになる。

  • IP アドレスの引き継ぎ

    • MAC アドレスをもとに通信を行っているため、キャッシュを削除するために他のサーバのARPテーブルを更新する必要がある
      • gratuitous ARP(GARP)
      • 私の IP アドレスと MAC アドレスはこれです、と他のサーバへ通知するためのもの
