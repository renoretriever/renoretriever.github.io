LAN アクセス制御

  • CSMA/CD 方式

    • 回線に信号が流れていないときだけパケットを送信
    1. 各ステーションが伝送媒体が使用中かを調べ、使用中でなければデータを送出する
    2. 伝送路上でフレームの衝突(コリジョン)が発生したときに、ランダムな時間だけ待ってから再送を試みる
    3. ジャム信号を一定時間送信し、衝突の発生をすべてステーションに検知させる
    4. ランダムな待ち時間(バックオフ時間)経過後に、再び 1 から送信手続きを開始
    • 単位時間あたりの送出フレーム数が増すと衝突の頻度が上がり、通信のスループットが低下するという弱点がありますが、比較的単純なアルゴリズムで制御できるため、現在は LAN のアクセス制御の主流
    • コリジョン減少
    • 構成
      • CS(Carrier Sense : キャリアセンス)
        • ケーブル上に信号が流れていないこと(アイドル状態)を確認する
      • MA(Multiple Access : 多重アクセス)
        • すべてのホストが平等に送信権利を持ち、ケーブルを共有する
      • CD(Collision Detection : 衝突検出)
        • 衝突を検出すると、ジャム信号を送信してランダムな時間待機(バックオフ)
    • 早い者勝ち
      • 優先順位など無い

  • トークンパッシング方式

    • 送信を開始するためにはネットワーク内を巡回する トークン を取得する必要ある
    • リング状の LAN に適していて、衝突が発生しないのでスループットが低下しにくいというメリットがありますが、現在ではあまり使われない

  • TDMA(Time Division Multiple Access)

    • 主に無線通信で使わせるアクセス制御
    • 同一周波数を短い時間単位で分割し、通信局ごとに切り替えてゆくことで、同一周波数で複数の通信局が同時に通信を行なうことが可能
    • タイムスロット(通信のタイミング)の割り当てが行われたあとには、基本的に衝突は発生しない
