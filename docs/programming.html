プログラミング

基礎

  • ライブラリー

    • 複数のオブジェクトファイルをまとめて収録したファイル
    • リンカーにライブラリを指定すると、その中から必要なオブジェクトだけを抽出し、それを他のオブジェクトと結合して EXE ファイルを生成してくれる
    • 標準ライブラリ関数
      • どのプログラムでも使える標準的な処理がライブラリとして提供されている
      • 例
        • stdio.h
          • fopen 関数
          • fgetc 関数
        • string.h
          • strcpy 関数
    • DLL(Dynamic Link Library)
      • プログラムの実行時に結合されるもの
      • メリット
    • スタティックリンクライブラリ
      • オブジェクトファイルの実体を格納し、EXE ファイルに直接結合してしまう形式のライブラリ

  • インタブリタ

    • 実行時にソースコードの内容を 1 行ずつ解釈して処理

  • 手続型プログラミング

    • 手続きを処理する順番に並べて命令文を記述するプログラム言語
    • 最も一般的な方式で、"C"，"COBOL"，"PASCAL"などのプログラム言語は手続き型に分類

  • 論理型プログラミング

    • 論理式の証明過程をプログラムで自動化するためのプログラム言語
    • 代表的なものに Prolog
    • プログラムに"事実"と"規則"を記述することで、言語の処理系がもつ導出原理によって"質問"に適合する事実を導き出すことが可能

  • 関数型プログラミング

    • 関数の定義によってプログラムを構成するプログラム言語
    • 命令を逐一実行するのではなく式を入れ子構造にし、式を計算結果を置き換えていくことで実行していく方式で、代表的なものに"Lisp"がある

  • エキスパートシステム

    • 大量の既知情報を処理することで、専門家の判断をエミュレートする人工知能システム
  • 単一値の置換

    • 単一の変数を置換する場合は、文字列の後ろに % と置換したい値を並べて記述

      valuedescription
      %s文字列として置換
      %d整数値として置換
      %f浮動小数点として置換
      %x16 進整数として置換

ソース・ファイルから実行可能なファイルができるまで

  1. ソースファイルの作成
    • ソースコード
      • 何らかのプログラミング言語で記述したプログラムのこと
      • ソースファイル
        • ソースコードをファイルとして保存したもの
  2. コンパイル により オブジェクトファイル(機械語) を作成
    • ネイティブ・コード(機械語)
      • CPU が解釈・実行できる形式のプログラム
      • コンパイルを実行することで得られる
    • コンパイラ

      • C 言語などの高水準言語で記述されたソースコードをネイティブ・コードに翻訳する機能を持ったプログラム

      • CPU の種類が異なれば、ネイティブコードの種類も異なる

      • ネイティブコードの生成

        • ソースコードの字句解析 => 構文解析 => 意味解析 => ネイティブコード生成

      • 例

        • C コンパイラ
          • C 言語で記述されたソースコードをネイティブコードに翻訳するコンパイラ

      • 実行例 : Borland C++ コンパイラ(bcc32.exe)

        bcc32 -W -c Sample1.c

        • 実行後は Sample1.obj が生成される

        • 実行可能な EXE ファイルを得るためには リンク という動作が必要

          ilink32 -Tpe -c -x -aa c0w32.obj Sample1.obj, Sample1.exe,,import32.lib cw32lib
          • -Tpe -c -x -aa
            • Windows 用の EXE ファイルを生成することを指定
          • c0w32.obj
            • スタートアップ
            • すべてのプログラムの先頭に結合する共通的な処理が記述されたオブジェクト・ファイル
            • Borland C++ によって提供されているプログラム
          • import32.lib cw32lib
            • ライブラリファイル
  3. リンク により実行ファイル(機械語)を作成
    • リンク
      • 複数のオブジェクトファイル(中間言語)を結合して 1 つの OS 上で起動可能な実行可能形式のファイルを作成
      • リンカはオブジェクトコード中から呼び出されるコードが収録された他のオブジェクトファイルやライブラリファイルなどを連結あるいは参照し、実行時に適切に呼び出せるよう繋ぎこむ
      • 静的リンク
        • リンカがコード中から参照されるライブラリ関数などすべての機械語プログラムを一つのファイルに格納し、単体で実行可能なファイルを生成
      • 動的リンク
        • 外部プログラムの連結は実行時に行われるため、リンカは実行ファイルにローダなど外部コードを呼び出す仕組みを組み込む
      • リンカー
        • リンクを実行するツール
  • ビルド
    • コンパイル => リンクを続けて行う意味
