WPA(Wi-Fi Protected Access)

基礎

  • WEP の代わりとして考えられた方式
  • 鍵を一定時間ごとに変更する技術を採用しているのが特徴
    • 通信を行う端末の MAC アドレスなどを基に、一時的な暗号化鍵を生成し、一定の通信量を超えると新たな鍵に変更される
  • 接続相手を認証するために IEEE802.1X が導入されている
    • 基本的には認証サーバによる暗号化鍵が必要
    • WPA-PSK を導入して上記を対応
      • PSK(Pre Shared Key) を利用
      • WEP と同様、事前に発行された鍵を使う
      • WPA では、WEP の脆弱性に対して RC4 128 bit の鍵を使用し、TKIP(Temporal Key Integrity Protocol) により動的に鍵を変更することで強度を高めている
      • 事前鍵共有モード(PSK) では 8 〜 63 文字のパスフレーズか、 64 桁の 16 進数を事前設定しておくことにより、IEEE802.1x 認証サーバを不要としている
      • WAP-TKIP では認証時のパケットからパスフレーズを解析する脆弱性が見つかっており、安全に使うには 21 文字以上の長いパスフレーズを使用するなどの注意が必要
      • 暗号技術が RC4 であるため万全じゃない
  • WPA / WPA2-EAP
    • IEEE802.1x 認証とプロトコル「TKIP」を組み合わせたもので、認証サーバの設置が必要となる
    • EAP(Extensible Authentication Protocol)
    • 暗号方式
      • EAP-TLS
        • 認証
          • ディジタル証明書による認証サーバとクライアントの相互認証
          • TLS のハンドシェイクプロトコルで用いられているサーバ証明書とクライアント証明書の検証によって、認証サーバとクライアントの相互認証を行うことができる
      • EAP-TTLS
        • 認証
          • ディジタル証明書による認証サーバとID, パスワード
      • etc

WPA2

  • WPA の問題点である暗号技術を改善し、AES を用いることで改善が図られている
  • 一般家庭では WPA2-PSK(WPA2 パーソナル) が使われることが多い
