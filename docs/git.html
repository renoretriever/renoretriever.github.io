Git

基本的な操作

  • リポジトリ初期化

    $ git init
Initialized empty Git repository in /Users/reno/gittest/.git/
    • .git ディレクトリ
      • リポジトリデータの格納

  • リポジトリの状態確認

    $ git status
On branch master

No commits yet

nothing to commit (create/copy files and use "git add" to track)

  • ステージ領域へファイルを追加

    $ touch README.md
$ git add README.md
$ git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

    new file:   README.md
    • これで Git リポジトリの管理対象になる
    • ステージ領域
      • コミットする前の一時領域

  • リポジトリの歴史を記録

    $ git commit -m "First commit"
[master (root-commit) 17d59e4] First commit
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 README.md

  • コミットログの確認

    • リポジトリにコミットされたログ

      $ git log

    • ファイルの差分を表示

      $ git log -p

  • 差分(ワークツリー、ステージ領域、最新コミット間) の表示

    • ワークツリーとステージ領域の差分確認

      $ git diff

    • ワークツリーと最新コミットの差分を確認

      $ git diff HEAD

ブランチ操作

  • ブランチを一覧表示

    $ git branch
* master
    • * がついているのが作業しているブランチ

ブランチを作成して切り替える

  • feature-A ブランチに切り替えコミット

    $ git checkout -b feature-A
Switched to a new branch 'feature-A'
$ git branch
* feature-A
  master
    • 下記と同じ操作

    1. feature-A ブランチの作成

      $ git branch feature-A

    2. ブランチの切り替え

      $ git checkout feature-A

  • master ブランチへ切り替え

    $ git checkout master
Switched to branch 'master'
$ git branch
feature-A
* master

  • 一つ前のブランチの切り替える

    $ git checkout -
Switched to branch 'feature-A'
$ git branch
* feature-A
  master

  • ブランチをマージ

    • マージ先の master に移動

      $ git checkout master
Switched to branch 'master'
$ git branch
  feature-A
* master

    • マージ処理

      $ git merge --no-ff feature-A
Merge made by the 'recursive' strategy.
 README.md | 1 +
 1 file changed, 1 insertion(+)

  • ブランチを視覚的に確認

    $ git log --graph

コミットを変更する操作

  • あるときまで戻す

    $ git reset --hard e2850144db3a15d8690d77a9d5a43d3a50d110d5

  • リモートに強制上書き

    $ git fetch origin
$ git reset --hard origin/master

  • git rm したファイルを元に戻す

    $ git reset HEAD <targetfile>
$ git checkout <targetfile>

  • git add の取り消し

    $ git reset HEAD

  • まだ git add していない変更を取り消す

    • 特定のファイルをHEADの状態に戻す

      $ git checkout HEAD <FILE_NAME>

    • 作業ツリーをHEADの状態に戻す

      $ git checkout HEAD .

  • git add したが、まだコミットしていない変更を取り消す

    • 作業ツリーとインデックスの特定のファイルを元に戻す

      $ git checkout HEAD <FILE_NAME>

    • 作業ツリーとインデックスを元に戻す

      $ git checkout HEAD .

リモートリポジトリへ送信

  • GitHub 側でリポジトリを作成

  • リモートリポジトリを登録

    $ git remote add origin git@github.com:renoretriever/git-tutorial.git

  • リモートリポジトリへ送信

    $ git push -u origin master
Counting objects: 11, done.
Delta compression using up to 4 threads.
Compressing objects: 100% (7/7), done.
Writing objects: 100% (11/11), 928 bytes | 132.00 KiB/s, done.
Total 11 (delta 0), reused 0 (delta 0)
To github.com:renoretriever/git-tutorial.git
 * [new branch]      master -> master
Branch 'master' set up to track remote branch 'master' from 'origin'.
    • -u
      • ローカルリポジトリの現在のブランチの upstream は origin リポジトリの master ブランチであることを設定
    • origin
      • リモートリポジトリ名
    • master
      • リモートリポジトリの master ブランチ

  • master ブランチ以外の feature-D ブランチを作成して、そこに push

    $ git checkout -b feature-D
Switched to a new branch 'feature-D'
$ git push -u origin feature-D
Total 0 (delta 0), reused 0 (delta 0)
remote:

remote:      https://github.com/renoretriever/git-tutorial/pull/new/feature-D
remote:
To github.com:renoretriever/git-tutorial.git
* [new branch]      feature-D -> feature-D
Branch 'feature-D' set up to track remote branch 'feature-D' from 'origin'.
    • ブラウザからブランチに feature-D があることを確認

リモートリポジトリの取得

  • リモートリポジトリの取得

    $ git clone git@github.com:renoretriever/git-tutorial.git
Cloning into 'git-tutorial'...
remote: Enumerating objects: 11, done.
remote: Counting objects: 100% (11/11), done.
remote: Compressing objects: 100% (7/7), done.
remote: Total 11 (delta 0), reused 11 (delta 0), pack-reused 0
Receiving objects: 100% (11/11), done.

    • 現在のブランチ情報を確認

      $ git branch -a
* master
  remotes/origin/HEAD -> origin/master
  remotes/origin/feature-D
  remotes/origin/master

  • リモートの feature-D ブランチをチェックアウト

    $ git checkout -b feature-D origin/feature-D
Branch 'feature-D' set up to track remote branch 'feature-D' from 'origin'.
Switched to a new branch 'feature-D'
$ git branch -a
* feature-D
  master
  remotes/origin/HEAD -> origin/master
  remotes/origin/feature-D
  remotes/origin/master
    • origin/feature-D
      • origin という名前のリモートリポジトリの feature-D ブランチをもとに手元のローカルリポジトリに feature-D ブランチを作成

  • 最新のデータを取得

    $ git pull

サブモジュールの追加

  • submodule-parent に submodule-child をサブモジュールとして追加してみる
  • GitHub 上で上記 2 リポジトリの作成

  • サブモジュールの追加

    $ git submodule add git@github.com:renoretriever/submodule-child.git

  • git add => commit => push するとサブモジュールに追加される

    • ブラウザ上で submodule-child @ 00e55f1 なのが表示される

      • 00e55f1 はコミット番号

    • 確認コマンド

      $ git submodule status

  • submodule-child での更新を submodule-parent 側で更新

    $ git submodule foreach git pull
