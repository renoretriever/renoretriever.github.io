ハッシュ

基礎

  • 特徴
    • ハッシュ関数を適用した結果から元のメッセージを推定できない
    • メッセージの長さに関係なく、ハッシュ値の長さが一定である
    • 同じハッシュ値になる別のメッセージを作成することが困難である

ハッシュアルゴリズム

  • ほぼ 1 回の計算でデータを探し出せる特性を持つ

  • 計算量

    • O(1)

  • メリット

    • データがどれだけ増えても一定時間で検索処理が終わること

  • デメリット

    • 同じデータに対しては同じハッシュになる
      • コリジョンの発生
      • => リスト構造でつなげる
      • => リハッシュの実施

ハッシュ関数

  • 通信相手は正しいのか、受け取った情報は改ざんされていないかを検査するために使われるのがハッシュ関数

  • 実装

    • SHA()Secure Hash Algorithm)

  • ハッシュ関数は、可変長のデータや文字列を入力すると一意に決まる固定長のデータ列（ハッシュ値）を出力するが、入力値が一文字でも違うと全く異なるハッシュ値になる特徴がある

  • TLSでは、ハッシュを偽造できないように共通鍵とハッシュを組み合わせた HMAC を通信データに付加することで完全性検証と認証を行っている

    • 認証付き暗号を使っている場合HMACは不要
  • RIPEMD(RACE Integrity Primitives Evaluation Message Digest)
    • 暗号学的ハッシュ関数
    • RIPEMD は MD4 の設計原理に基づいたものであり、SHA-1 と同程度のパフォーマンスを有している

  • 入力データから固定長のビット列(128bit, 160 bit)を出力する関数で、出力されたハッシュ値から元の値を算出できない

    • 元のデータを一部変更するとハッシュ値が大きく異なるという性質がある
    • 出力された値
      • ハッシュ値
      • メッセージダイジェスト

  • ハッシュ関数の性質

    • ハッシュ値から元の入力データの内容を推測できない
    • 元のデータの一部を変更するとハッシュ値が大きく異る
    • 同じハッシュ値になる入力データを見つけるのは困難

  • ハッシュ関数の応用

    • パスワードの保存
    • デジタル証明書
    • 改ざん検知
    • ワンタイムパスワード

  • 性質(下に行くにつれて困難)

    • 衝突発見困難性
      • ハッシュ値が一致する二つのメッセージの探索に要する計算量の大きさによる、探索の困難性
    • 第 2 原像計算困難性
      • データとそれに対するハッシュ値が与えられたとき、同じハッシュ値を生成するデータを求めることが計算量的に困難であること
    • 原像計算困難性
      • ハッシュ値が与えられたときに、元のメッセージの探索に要する計算量の大きさによる、探索の困難性
      • ハッシュ値の元のメッセージの探索に要する最大の計算量は 2 の 256 乗

  • 代表的なハッシュ関数

    • MD5
    • SHA
