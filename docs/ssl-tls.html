TLS/SSL 暗号化通信の仕組み

  • TLS/SSL ハンドシェイク(c: クライアント, s: サーバ)

    1. c => s : リクエスト送信
    2. s => c : サーバ証明書送付
    3. c : サーバ証明書から、サーバの公開鍵を取得
    4. c : 共通鍵を生成
    5. c : サーバの公開鍵で共通鍵を暗号化
      • 公開鍵暗号方式
    6. c => s : 暗号化された共通鍵を送付
    7. s : 暗号化された共通鍵を、サーバの秘密鍵で復号化し、共通鍵を取得
    8. s : 送信データを共通鍵で暗号化
    9. c : 受信したデータを共通鍵で復号化

  • セッションキャッシュ

    • TLS ハンドシェイク時に作成されたクライアント/サーバで利用される共通のセッション ID を共有メモリにキャッシュし、次回の TLS ハンドシェイクを省略できる

  • セッションチケット

    • セッションキャッシュはセッション情報を共有メモリにキャッシュするため、複数台のサーバ間でセッション情報をキャッシュできない
    • セッションチケットを利用すると、セッション情報を暗号化しチケットとしてクライアントに送信。クライアントはこのチケットをサーバに送信することで前回の TLS セッションを再開できる
